3.3 Copias de seguridad

Duplicar datos en una ubicación segura para poder recuperarlos si algo falla

¿Por qué son importantes?

🛡️
Prevenir pérdidas
Fallos técnicos, errores humanos o malware pueden borrar datos
🏢
Continuidad del negocio
Reduce el tiempo de inactividad ante un incidente
⚖️
Requisitos legales
Algunas regulaciones obligan a conservar ciertos datos
🔐
Defensa ante ransomware
Permite restaurar sin pagar el rescate a los atacantes

Aspectos clave al hacer un backup

⏱ Frecuencia
Según qué tan seguido cambian los datos: diaria, semanal o en tiempo real
💾 Medio
Disco externo, cinta, servidor de red o nube
📍 Ubicación
Siempre fuera del sitio principal (nube o ubicación remota)
🔒 Seguridad
Cifrado, controles de acceso y protección física
🤖 Automatización
Reduce errores humanos y garantiza regularidad
✅ Verificación
Probar periódicamente que los datos se pueden restaurar
Windows Métodos de copia de seguridad
1. Historial de archivos
Panel de control → Sistema y seguridad → Historial de archivos. Selecciona unidad, activa y personaliza carpetas y frecuencia.
Restaurar: Historial de archivos → Restaurar archivos personales
2. Copia de seguridad y restauración
Panel de control → Copia de seguridad y restauración (Windows 7). Permite elegir qué respaldar y programar copias automáticas.
Restaurar: botón "Restaurar mis archivos"
3. OneDrive (nube)
Inicia sesión con cuenta Microsoft. Selecciona carpetas a sincronizar. Los archivos quedan accesibles desde cualquier dispositivo.
Ubuntu Métodos de copia de seguridad
1. Deja Dup (gráfico)
App "Copias de seguridad". Elige destino (local, externo o Google Drive), carpetas y frecuencia.
Instalar: sudo apt install deja-dup
2. rsync (terminal)
Sincroniza carpetas origen → destino. Con --delete elimina archivos que ya no existen en el origen.
Automatizar con cron: 0 2 * * * rsync -av ...
3. Rclone (nube)
Soporta Google Drive, Dropbox y más. Se configura con rclone config y sincroniza con rclone sync.
Instalar: sudo apt install rclone
