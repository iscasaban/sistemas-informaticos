3.5 Tareas automáticas

Programar tareas repetitivas para que se ejecuten solas, sin intervención manual

El formato cron (Ubuntu)

Estructura de una línea cron
*Minuto
0–59
*Hora
0–23
*Día mes
1–31
*Mes
1–12
*Día semana
0–7
0 3 * * * rsync -av /documentos /respaldo
→ Ejecuta rsync todos los días a las 3:00 a.m.
Windows Herramientas de automatización
🗓 Programador de tareas Gráfico
Herramienta principal para programar procesos. Acceso buscando "Programador de tareas" o con taskschd.msc.
✔ Programar por horario o por evento
✔ Configurar condiciones y desencadenantes
✔ Ejecutar scripts, programas o comandos
Ejemplo: Crear tarea básica → desencadenante Semanal → acción Iniciar programa → guardar
💻 PowerShell + Programador Script
Crea un script .ps1 y prográmalo en el Programador de tareas como acción.
Remove-Item -Path "C:\Temp\*" -Recurse -Force
→ limpia archivos temporales
Acción en el programador: powershell.exe -File "C:\ruta\script.ps1"
📄 Archivos por lotes (.bat) Script
Ejecutan comandos de Windows en secuencia. Se programan también desde el Programador de tareas.
xcopy C:\Documentos D:\Backup /E /I
→ copia de seguridad de Documentos
Apagado automático: shutdown /s /t 0 programado a hora específica
Ubuntu Herramientas de automatización
⏰ Cron Terminal
Servicio para tareas recurrentes. Edita la configuración con crontab -e.
0 3 * * *backup diario a las 3:00 a.m.
0 2 * * 0tarea cada domingo a las 2:00 a.m.
0 2 * * 0 rm -rf /tmp/*
→ limpia /tmp/ cada domingo
Editar crontab: crontab -e → añade una línea por tarea
🕐 At Terminal
Para tareas únicas en una hora concreta (no recurrentes). Requiere instalar el servicio.
sudo apt install at
sudo systemctl enable atd
echo "shutdown now" | at 23:00
→ apaga el sistema a las 23:00
🖥 GNOME Schedule Gráfico
Interfaz gráfica para gestionar cron sin escribir comandos. Opcional, no viene por defecto.
sudo apt install gnome-schedule
✔ Crea y edita tareas recurrentes visualmente
✔ Supervisa tareas programadas en ejecución
