5.7 Redes de área extensa (WAN)

Conexiones seguras entre ubicaciones remotas mediante VPN y enrutamiento WAN.

¿Qué es una WAN? Una red de área extensa (Wide Area Network) conecta redes locales distantes entre sí. Las VPN son la herramienta principal para hacerlo de forma segura cifrando el tráfico.

Windows Redes WAN y VPN

Configurar VPNGUI
1
Configuración → Red e Internet → VPN
2
Clic en Agregar una conexión VPN
3
Introduce: nombre, dirección del servidor y tipo de protocolo
4
Guarda y haz clic en Conectar
Protocolos compatibles
PPTP L2TP/IPsec IKEv2 SSTP
VPN y rutas WANCLI
Añadir VPN con PowerShell
Add-VpnConnection -Name "MiVPN" ` -ServerAddress "vpn.example.com" ` -TunnelType L2tp ` -EncryptionLevel Required ` -AuthenticationMethod Pap ` -L2tpPsk "ClaveSecreta"
Conectar a la VPN
Connect-VpnConnection -Name "MiVPN"
Enrutar tráfico por la WAN
route add 10.0.0.0 MASK 255.255.255.0 192.168.1.254

Ubuntu Redes WAN y VPN

Configurar VPNGUI
1
Configuración → Red → VPN
2
Clic en + para agregar nueva VPN
3
Selecciona el tipo: PPTP, L2TP o OpenVPN
4
Introduce servidor, autenticación y credenciales → Guardar
OpenVPN y WireGuardCLI
OpenVPN
sudo apt install openvpn
sudo openvpn --config /ruta/a/archivo.ovpn
ip addrVerificar conexión
WireGuard (moderno y rápido)
sudo apt install wireguard
sudo wg-quick up wg0Iniciar conexión
Enrutar tráfico por la WAN
sudo ip route add 10.0.0.0/24 via 192.168.1.254 dev eth0

Rutas WAN persistentes en Ubuntu (Netplan)

Ubuntu/etc/netplan/01-netcfg.yaml
network: version: 2 ethernets: eth0: dhcp4: no addresses: [192.168.1.100/24] gateway4: 192.168.1.1 routes: - to: 10.0.0.0/24 via: 192.168.1.254
sudo netplan applyAplica la configuración
