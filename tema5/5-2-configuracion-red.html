5.2 Configuración de red

Parámetros esenciales para conectar un dispositivo a la red correctamente.

Parámetros de configuración

🖥️ Dirección IP
Identifica al dispositivo en la red. Puede ser estática (manual) o dinámica (DHCP).
🎭 Máscara de subred
Separa la parte de red de la parte de host. Ej: 255.255.255.0
🚪 Puerta de enlace
Dispositivo que conecta la red local con el exterior (internet).
🔍 DNS
Traduce nombres de dominio (como google.com) a direcciones IP.
⚡ DHCP
Servidor que asigna IP automáticamente al conectarse a la red.
🔧 Avanzado
NAT, VLANs, QoS — para redes más complejas y segmentadas.
Windows Configuración gráfica GUI
1
Inicio → Configuración → Red e Internet
2
Estado → Cambiar opciones de adaptador
3
Clic derecho en la red → Propiedades
4
Selecciona TCP/IPv4 → Propiedades
5
Introduce IP, máscara, gateway y DNS → Aceptar
Windows Línea de comandos CLI
Asignar IP estática
netsh interface ip set address name="Ethernet" static 192.168.1.10 255.255.255.0 192.168.1.1
Configurar DNS
netsh interface ip set dns name="Ethernet" static 8.8.8.8
Verificar configuración
ipconfig /allVer toda la config de red
ping 8.8.8.8Comprobar conectividad
Ubuntu Configuración gráfica (NetworkManager) GUI
1
Icono de red → Configuración de red
2
Selecciona la conexión (Ethernet o Wi-Fi)
3
Pestaña IPv4 → Método: Manual
4
Introduce IP, máscara, gateway y DNS
5
Clic en Aplicar → verificar con ip a
Ubuntu Netplan (desde Ubuntu 18.04) CLI
Editar archivo de configuración
sudo nano /etc/netplan/01-netcfg.yaml
Contenido del archivo .yaml
network: version: 2 ethernets: eth0: dhcp4: no addresses: [192.168.1.10/24] gateway4: 192.168.1.1 nameservers: addresses: [8.8.8.8, 8.8.4.4]
Aplicar cambios
sudo netplan applyActiva la nueva config
ip aVerificar interfaces
💡 Recuerda: Sustituye eth0 por el nombre real de tu interfaz de red. Para conocerlo, ejecuta ip a antes de editar el archivo YAML.
