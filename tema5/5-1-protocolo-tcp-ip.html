5.1 Protocolo TCP/IP

El conjunto de protocolos que permite la comunicación entre dispositivos en cualquier red

Las 4 capas del modelo TCP/IP

🌐 Capa de aplicación
Servicios directos al usuario: web, email, archivos
HTTP FTP SMTP DNS SSH DHCP SNMP
🚚 Capa de transporte
Transmisión fiable de datos entre dispositivos
TCP UDP DCCP
🗺 Capa de internet
Direccionamiento y encaminamiento de paquetes
IP ICMP IPSEC IGMP
🔌 Capa de enlace de datos
Transmisión física a través del medio (cable, Wi-Fi)
Ethernet Wi-Fi ARP PPP L2TP

TCP vs UDP — ¿cuál usar?

TCP — orientado a conexión
Garantía: entrega confirmada y en orden
Mecanismo: confirmaciones y retransmisiones
Velocidad: más lento (mayor control)
Uso: web, email, transferencia de archivos
UDP — sin conexión
Garantía: no garantiza entrega ni orden
Mecanismo: envío directo sin confirmación
Velocidad: más rápido (menor control)
Uso: streaming, videojuegos, DNS

Flujo de datos en TCP/IP

① Inicio
Capa de aplicación
La app solicita enviar datos. Se prepara el mensaje.
② Segmentación
Capa de transporte
Los datos se dividen en paquetes numerados (TCP establece la conexión).
③ Direccionamiento
Capa de internet
Se asignan IPs de origen y destino. Los paquetes se encapsulan.
④ Transmisión
Capa de enlace
Los datagramas se convierten en tramas y viajan por el medio físico.
⑤ Recepción
Desencapsulación
El receptor reconstruye los datos subiendo por las capas.
⑥ Entrega
Aplicación destino
Los datos llegan a la app receptora. Ciclo completado.

IPv4 vs IPv6

IPv4
Formato: 4 números (192.168.1.1)
Direcciones: ~4.300 millones
Estado: en uso actualmente, agotándose
IPv6
Formato: 8 grupos hexadecimales
Direcciones: prácticamente ilimitadas
Ventajas: mejor enrutamiento y seguridad
