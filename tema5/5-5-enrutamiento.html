5.5 Enrutamiento

Seleccionar el camino más eficiente para que los datos viajen entre redes distintas.

¿Para qué sirve? El enrutamiento permite gestionar redes locales, acceder a múltiples subredes y usar conexiones VPN. Sin rutas correctas, los paquetes no llegan a su destino.
Windows Comando route CLI
Ver tabla de enrutamiento
route print
Muestra: destinos de red, máscaras, gateways e interfaces asociadas.
Agregar una ruta
route add [destino] MASK [máscara] [gateway]
Ejemplo — subred 192.168.2.0/24 via 192.168.1.1:
route add 192.168.2.0 MASK 255.255.255.0 192.168.1.1
Ruta permanente (persiste al reiniciar)
route -p add 192.168.2.0 MASK 255.255.255.0 192.168.1.1
Eliminar una ruta
route delete 192.168.2.0
Ejemplo completo — ruta estática
route add 10.0.0.0 MASK 255.255.255.0 192.168.1.254 route print
Ubuntu Comando ip route CLI
Ver tabla de enrutamiento
ip route show
Salida típica:
default via 192.168.1.1 dev eth0
192.168.1.0/24 dev eth0 proto kernel scope link
Agregar una ruta (temporal)
sudo ip route add [destino] via [gateway] dev [interfaz]
Ejemplo — subred 10.0.0.0/24:
sudo ip route add 10.0.0.0/24 via 192.168.1.1 dev eth0
Eliminar una ruta
sudo ip route del 10.0.0.0/24
Ruta persistente — editar Netplan
sudo nano /etc/netplan/01-netcfg.yaml

Rutas persistentes en Ubuntu (Netplan)

Ubuntu Archivo /etc/netplan/01-netcfg.yaml
network: version: 2 ethernets: eth0: dhcp4: no addresses: [192.168.1.100/24] gateway4: 192.168.1.1 routes: - to: 10.0.0.0/24 via: 192.168.1.1
Aplicar cambios
sudo netplan applyActiva la configuración
ip route showVerificar que la ruta existe
