4.4 Servicios y procesos

Entender qué se está ejecutando en el sistema y cómo controlarlo

Proceso vs Servicio — ¿en qué se diferencian?

⚙️ Proceso
Qué es: un programa en ejecución
Quién lo inicia: el usuario (abrir una app)
Visibilidad: interacción directa con el usuario
Ciclo de vida: se abre y cierra según necesidad
Ejemplo: el navegador, un editor de texto
🔧 Servicio
Qué es: proceso en segundo plano
Quién lo inicia: el sistema al arrancar
Visibilidad: sin interfaz, el usuario no lo ve
Ciclo de vida: activo mientras el sistema esté encendido
Ejemplo: gestión de red, impresión, actualizaciones
Windows 10 Gestión de procesos y servicios
📊 Administrador de tareas — procesos
Gráfico
Acceso: Ctrl + Shift + Esc
Pestaña Procesos → lista con uso de CPU, memoria, disco y red
→ seleccionar proceso → Finalizar tarea
Pestaña Rendimiento → gráficos en tiempo real
Servicios desde el Administrador: pestaña Servicios → clic derecho para iniciar, detener o reiniciar
🔧 Consola de servicios
Gráfico
Acceso: Win + R → escribe services.msc
→ clic derecho sobre un servicio → Iniciar / Detener / Reiniciar
→ Propiedades → Tipo de inicio: Automático, Manual o Deshabilitado
Ubuntu Gestión de procesos y servicios
⚙️ Gestión de procesos
Terminal
Listar todos los procesos:ps aux
Vista en tiempo real (interactiva):top
Versión mejorada de top:htop
Finalizar proceso por su ID:kill PID
Forzar cierre si no responde:kill -9 PID
GUI: Monitor del sistema → equivalente al Administrador de tareas
🔧 Gestión de servicios — systemctl
Terminal
Ver estado:sudo systemctl status servicio
Iniciar:sudo systemctl start servicio
Detener:sudo systemctl stop servicio
Reiniciar:sudo systemctl restart servicio
Activar al inicio:sudo systemctl enable servicio
Desactivar del inicio:sudo systemctl disable servicio
systemd es el sistema de gestión de servicios en Ubuntu. El comando principal es systemctl
