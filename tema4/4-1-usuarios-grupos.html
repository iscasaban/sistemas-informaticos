4.1 Usuarios y grupos

Controlar quién accede al sistema y qué puede hacer en él

¿Por qué es importante gestionarlos?

🔐
Seguridad
Control de acceso por roles: cada usuario solo hace lo que tiene autorizado
🗂
Organización
Asignar permisos a grupos, no uno a uno — menos errores, más rapidez
📋
Auditoría
Registrar acciones de cada usuario facilita revisiones de seguridad
⚙️
Eficiencia
Gestionar permisos por grupos similares simplifica la administración
Windows 10 Usuarios y grupos
Crear usuario
Gráfico
Configuración → Cuentas → Familia y otras personas
→ Agregar otra persona a este equipo
→ Introduce nombre, contraseña y pregunta de seguridad
Asignar usuario a grupo
Gráfico
Win + X → Administración de equipos
→ Usuarios y Grupos Locales → Usuarios
→ Clic derecho en usuario → Propiedades → Miembro de
Grupos comunes
Administradores
Control total del sistema
Usuarios
Permisos estándar, sin cambios críticos
Invitados
Acceso muy limitado, uso temporal
Ubuntu Usuarios y grupos
Crear usuario
Gráfico
Configuración del sistema → Detalles → Usuarios
→ Desbloquear → botón + → Introduce nombre y tipo de cuenta
sudo adduser nombre_usuariocrea usuario por terminal
Crear grupo y asignar usuario
Terminal
sudo groupadd nombre_grupocrear grupo nuevo
sudo usermod -aG grupo usuarioañadir usuario al grupo
Ejemplo: sudo usermod -aG sudo juan → da a "juan" permisos de administrador
Grupos comunes
sudo
Privilegios de root con sudo
adm
Accede a archivos de registro
dialout
Puertos serie y USB
