4.2 Permisos en un sistema operativo

Controlar qué puede hacer cada usuario sobre cada archivo o carpeta

Tipos de permiso (comunes a ambos sistemas)

R
Lectura
Ver el contenido de un archivo o listar un directorio
W
Escritura
Modificar, crear o eliminar archivos en un directorio
X
Ejecución
Ejecutar un programa o acceder al contenido de un directorio

¿A quién se asignan los permisos?

👤
Propietario
El usuario dueño del archivo
👥
Grupo
Usuarios con permisos compartidos
🌐
Otros
El resto de usuarios del sistema

Notación numérica chmod (Ubuntu)

Cada dígito representa los permisos de un nivel: propietario / grupo / otros
7Propietarior + w + x
5Grupor + x
5Otrosr + x
sudo chmod 755 archivo
→ Propietario puede todo (7 = rwx) · Grupo y otros solo leer y ejecutar (5 = r-x)
Windows 10 Permisos (ACL)
Asignar permisos
Gráfico
Clic derecho en archivo/carpeta → Propiedades
→ Pestaña Seguridad → Editar
→ Selecciona usuario/grupo → marca o desmarca permisos
→ Aplicar → Aceptar
Permisos comunes
Control total — cualquier operación, incluido cambiar permisos
Modificar — cambiar el contenido
Lectura y ejecución — abrir o ejecutar
Escritura — cambiar contenido o añadir archivos
Ubuntu Permisos (rwx)
Ver permisos
Terminal
ls -lmuestra permisos → ej: -rwxr-xr--
Lectura: 3 grupos de 3 caracteres = propietario / grupo / otros
Cambiar permisos y propietario
Terminal
sudo chmod 755 archivocambia permisos
sudo chown juan archivo.txtcambia propietario
sudo chown juan:grupo archivo.txtcambia propietario y grupo
Asignar permisos (GUI)
Gráfico
Clic derecho → Propiedades → pestaña Permisos
→ Menús desplegables para Propietario, Grupo y Otros
→ Clic en Cerrar para aplicar
