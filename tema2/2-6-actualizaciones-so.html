2.6 Actualizaciones de los SO

Por qué actualizar · Peligros · Ciclo de vida · LTS

Idea clave: Mantener el SO actualizado es fundamental para garantizar seguridad, estabilidad y compatibilidad. No hacerlo expone el equipo a ataques y problemas graves.
✅

¿Por qué es importante actualizar?

🔒

Seguridad

Corrigen vulnerabilidades que los hackers pueden explotar. Sin updates → más riesgo de virus, malware y ransomware.

⚡

Estabilidad y rendimiento

Mejoran el funcionamiento del software y corrigen errores que ralentizan o bloquean el sistema.

🔌

Compatibilidad

Garantizan que el SO pueda ejecutar nuevo software, hardware y servicios en red.

✨

Nuevas funcionalidades

Incorporan características que mejoran la experiencia y productividad del usuario.

🔧

Cómo actualizar

🪟 Windows GUI

Ve a:

Configuración → Actualización y seguridad → Windows Update

Haz clic en Buscar actualizaciones e instala todo lo disponible.

🐧 Ubuntu CLI

Desde el terminal:

sudo apt update sudo apt upgrade

O por GUI: Software y actualizaciones → Buscar actualizaciones

⚠️

Peligros de NO actualizar

🦠

Vulnerabilidades de seguridad

Las brechas conocidas quedan abiertas. Los atacantes las explotan para robar datos, instalar malware o ransomware.

🔌

Incompatibilidad con software y hardware

El SO desactualizado puede no ejecutar nuevas versiones de apps o drivers, causando fallos y ralentizaciones.

💥

Problemas de estabilidad y rendimiento

Los bugs sin corregir pueden provocar bloqueos, cuelgues e inestabilidad general del sistema.

🚫

Pérdida de soporte técnico

El fabricante deja de ofrecer parches y ayuda para versiones antiguas. Si hay un problema, no habrá solución oficial.

🎯

Exposición a ataques dirigidos

Los atacantes buscan específicamente SO desactualizados con vulnerabilidades conocidas y públicas.

🔄

Ciclo de vida del software

Todo SO pasa por estas fases. Conocerlas ayuda a saber cuándo actualizar o cambiar de versión.

🛠️

Desarrollo

Se crea y prueba el SO. Se corrigen errores antes del lanzamiento.

🚀

Lanzamiento inicial

Primera versión pública. Parches rápidos ante errores iniciales.

✅

Soporte activo

Updates regulares: nuevas funciones, seguridad y corrección de bugs.

🔧

Soporte extendido

Solo actualizaciones de seguridad y errores críticos. Sin nuevas funciones.

💀

Fin de vida (EOL)

Sin soporte de ningún tipo. Muy vulnerable. Hay que migrar.

EOL = End Of Life — Fin del ciclo de soporte

LTS

¿Qué es el LTS? (Long Term Support)

LTS = versiones de software diseñadas para recibir soporte a largo plazo, generalmente 5 a 10 años. Muy comunes en distribuciones Linux como Ubuntu.

🏛️ Estabilidad

Diseñadas para ser muy estables. Ideales para entornos de producción y servidores.

📅 Soporte prolongado

Parches de seguridad garantizados durante años. Sin preocupación por obsolescencia a corto plazo.

🔒 Actualizaciones de seguridad

Reciben fixes de seguridad y corrección de errores críticos sin cambios drásticos.

🐢 Menos cambios drásticos

No incluyen las últimas novedades. El objetivo es la estabilidad, no la vanguardia tecnológica.

⚖️ LTS vs versión normal

Aspecto LTS Long Term Support Normal Versión estándar
Período de soporte 5 – 10 años 1 – 2 años
Nuevas funciones Pocas (prioridad: estabilidad) Sí, frecuentes
Ideal para Servidores, empresas, producción Uso doméstico, desarrollo
Riesgo de cambio Bajo Medio-alto
Ejemplo Ubuntu Ubuntu 22.04 LTS Ubuntu 23.10
