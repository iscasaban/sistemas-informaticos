2.3 Virtualización

Tipos · Ventajas · Componentes clave · Desafíos

¿Qué es? La virtualización consiste en crear una versión virtual de algo real (servidor, SO, almacenamiento o red) usando software, de forma que varios sistemas puedan convivir en un único hardware físico.
TIPOS

Tipos de virtualización

🗄️ De servidores

Múltiples SO en un único servidor físico mediante Máquinas Virtuales (VM).

  • Mejor uso del hardware.
  • Menos servidores físicos necesarios.
  • Facilita recuperación ante desastres.

🖥️ De escritorios

Escritorios virtuales accesibles desde cualquier dispositivo o lugar.

  • Entorno de trabajo consistente.
  • Datos centralizados → más seguridad.
  • Administración simplificada.

💾 De almacenamiento

Varios dispositivos físicos agrupados en una única unidad virtual.

  • Mejor gestión del espacio.
  • Escalable y flexible.
  • Administración centralizada.

🌐 De redes

Redes virtuales que se comportan como físicas.

  • Segmentación y aislamiento de redes.
  • Mejor uso de recursos de red.
  • Configuración más sencilla.
COMPONENTES

Componentes clave

🔧 El Hipervisor (VMM — Virtual Machine Monitor)

Software que crea y gestiona las máquinas virtuales. Es la capa de abstracción entre el hardware y los SO virtuales. Hay dos tipos:

Tipo 1 — Bare Metal

Se ejecuta directamente sobre el hardware. Sin SO anfitrión.

VMware ESXi Microsoft Hyper-V

Tipo 2 — Hosted

Se ejecuta sobre un SO anfitrión ya instalado.

VMware Workstation Oracle VirtualBox

VENTAJAS

Ventajas de la virtualización

⚡

Eficiencia de recursos

Más VMs en menos hardware físico.

💰

Reducción de costes

Menos hardware y menor consumo energético.

📈

Flexibilidad y escalabilidad

Crear o eliminar entornos fácilmente según necesidades.

🔄

Recuperación ante desastres

Copia y restauración rápida de VMs completas.

🛠️

Gestión simplificada

Administración centralizada de todos los recursos.

DESAFÍOS

Desafíos de la virtualización

🔓

Seguridad

Mayor superficie de ataque al gestionar múltiples capas de software en un mismo hardware.

📊

Gestión de recursos

Si no se gestionan bien las VMs, pueden sobrecargar el hardware físico y degradar el rendimiento.

🔌

Compatibilidad

Algunas aplicaciones antiguas (software heredado) pueden no funcionar correctamente en entornos virtualizados.

En resumen: La virtualización mejora la eficiencia, reduce costes y facilita la gestión. Sus ventajas superan con creces los desafíos, convirtiéndola en una tecnología esencial en cualquier infraestructura moderna.
