2.2 Clasificación de los Sistemas Operativos

Cómo se clasifican según distintos criterios clave

Idea clave: Los SO se pueden clasificar de muchas formas distintas según el tipo de hardware, cuántos usuarios los usan, qué tareas hacen, cómo interactúan con el usuario, su modo de uso y su licencia.

🖥️  Criterio 1 — Según el tipo de ordenador

🏠 Escritorio

Para PC doméstico u oficina.

Windows macOS Ubuntu

🗄️ Servidor

Gestiona recursos para múltiples usuarios en red.

Windows Server CentOS Red Hat

📱 Móvil

Diseñado para smartphones y tablets.

Android iOS

⏱️ Tiempo real (RTOS)

Respuesta inmediata y predecible. Sistemas embebidos.

FreeRTOS VxWorks QNX

🏢 Mainframe

Grandes sistemas empresariales con alta disponibilidad y procesamiento masivo. Ej.: z/OS (IBM)

👤 Criterio 2 — Nº de usuarios

1️⃣ Monousuario

Un solo usuario a la vez.

MS-DOS

👥 Multiusuario

Varios usuarios simultáneos compartiendo recursos.

Unix Linux Windows Server

📋 Criterio 3 — Nº de tareas

1️⃣ Monotarea

Solo una tarea a la vez.

MS-DOS

📦 Multitarea

Múltiples procesos ejecutándose a la vez. Gestiona el tiempo de CPU.

Windows macOS Linux

🖱️  Criterio 4 — Según la interfaz de usuario

CLI Línea de comandos

  • El usuario escribe comandos en una terminal.
  • Más técnica pero muy potente.
  • Preferida por usuarios avanzados.

Ej.: MS-DOS Bash (Unix/Linux)

GUI Interfaz gráfica

  • Ventanas, iconos y menús visuales.
  • Más intuitiva y accesible.
  • Estándar en sistemas modernos.

Ej.: Windows macOS Ubuntu (GNOME/KDE)

⚙️  Criterio 5 — Según el modo de uso
Tipo Descripción Ejemplos
Tiempo compartido Múltiples usuarios comparten CPU. Cada uno cree tener su propio procesador. Unix Linux Multics
Tiempo real Respuesta predecible y rápida. Crítico en sistemas de precisión. RTLinux VxWorks QNX
Distribuido Varios ordenadores independientes que parecen uno solo al usuario. Google Fuchsia Plan 9
📄  Criterio 6 — Según la licencia

🔓 Código abierto (Open Source)

  • Código fuente público.
  • Cualquiera puede ver, modificar y distribuir.
  • Comunidad activa de desarrollo.

Linux FreeBSD Android (AOSP)

🔒 Código cerrado (Privativo)

  • Código fuente no disponible al público.
  • Propiedad de una empresa.
  • Requiere licencia de uso.

Windows macOS iOS
