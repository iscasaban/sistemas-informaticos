1.1 Arquitectura Von Neumann

Propuesta por John von Neumann en 1945. Modelo base de casi todos los ordenadores modernos.

Idea clave: datos e instrucciones se almacenan en la misma memoria. Esto permite que los programas puedan modificarse durante su ejecución.

Componentes principales

🧠
CPU
Ejecuta instrucciones. Se divide en Unidad de Control (CU) y ALU.
💾
Memoria
Almacena datos e instrucciones. Organizada en celdas con direcciones.
⌨️
Entrada / Salida
Comunica el sistema con el exterior. Teclado, pantalla, discos…
🔌
Bus de comunicación
Canal que conecta CPU, memoria y dispositivos E/S.
🎛️
Unidad de Control (CU)
Interpreta y ejecuta las instrucciones del programa.
➕
ALU
Realiza operaciones matemáticas (suma, resta) y lógicas (AND, OR).

Ciclo de instrucción (cómo funciona)

1
Búsqueda
Fetch
La CPU lee una instrucción desde la dirección de memoria indicada.
2
Decodificación
Decode
La Unidad de Control interpreta la instrucción y decide qué hacer.
3
Ejecución
Execute
Se realiza el cálculo, movimiento de datos o comunicación con E/S.
4
Almacenamiento
Store
Los resultados se guardan en memoria o se envían a un dispositivo de salida.
🔁 Este ciclo se repite para cada instrucción del programa, de forma continua.
✅ Ventajas
✓
Simplicidad: estructura unificada, diseño de hardware menos complejo.
✓
Flexibilidad: los programas se pueden almacenar, modificar y ejecutar desde la misma memoria.
✓
Economía: memoria compartida para datos e instrucciones reduce costes de hardware.
⚠️ Limitaciones
!
Cuello de botella: CPU y memoria comparten un único canal de comunicación → ralentiza sistemas de alto rendimiento.
!
Riesgo de corrupción: datos e instrucciones comparten espacio de memoria → un error puede sobrescribir instrucciones.
🚀 Optimizaciones modernas
→
Memoria caché: reduce el cuello de botella guardando datos de uso frecuente cerca de la CPU.
→
Arquitecturas paralelas: varios núcleos o CPUs procesan múltiples instrucciones a la vez.

Resumen: partes de la CPU

ComponenteFunción
Unidad de Control (CU)Interpreta y coordina la ejecución de instrucciones
ALURealiza operaciones matemáticas y lógicas
RegistrosMemoria ultrarrápida interna que guarda datos temporales
Contador de programaGuarda la dirección de la próxima instrucción a ejecutar
InterrupcionesPausan la CPU para atender eventos externos (teclado, red…)
