6.6 Gestión de dominios

Administración centralizada de usuarios, equipos y políticas en una red empresarial.

🏢
Dominio
Grupo de equipos y usuarios gestionados de forma centralizada.
🖥️
Controlador de dominio
Servidor que autentica usuarios y aplica políticas en la red.
📋
Políticas de grupo
Reglas que se aplican automáticamente a usuarios y equipos del dominio.

Windows Active Directory

🏢 Instalar Active Directory (AD DS)GUI
1
Abre el Administrador del servidor
2
Agregar roles y características → selecciona AD DS (Servicios de dominio de Active Directory)
3
Completa el asistente y promueve el servidor como controlador de dominio
4
Define el nombre del dominio, p. ej. empresa.local
5
Administra usuarios y grupos desde el Administrador de Active Directory
💡 Ventaja clave: con Active Directory puedes controlar desde un único punto todos los equipos, usuarios, contraseñas y políticas de la red.
📋 Políticas de grupo (GPO)CLI / GUI
Acceso al editor
gpedit.mscEditor de políticas locales
Ruta: Configuración del equipo → Configuración de Windows → Seguridad → Políticas de cuenta
¿Qué se puede controlar con GPO?
✓
Requisitos de contraseñas (longitud, complejidad)
✓
Bloqueo de aplicaciones o acceso al sistema
✓
Configuración automática de escritorios y software
✓
Restricción de dispositivos USB u otros periféricos

Ubuntu Samba DC y realmd

🏢 Controlador de dominio con SambaCLI
Instalación
sudo apt install samba smbclient
Configuración en /etc/samba/smb.conf
[global] workgroup = EMPRESA security = user domain logons = yes logon path = \\%N\profiles\%U logon drive = H:
Reiniciar servicios
sudo systemctl restart smbd nmbd
Gestionar usuarios del dominio
Listar usuarios del dominio:
wbinfo -u
Listar grupos del dominio:
wbinfo -g
Asignar propietario del dominio:
sudo chown dominio\usuario /ruta/carpeta
🔗 Unir Ubuntu a un dominio existenteCLI
1. Instalar herramientas
sudo apt install realmd sssd sssd-tools adcli
2. Descubrir el dominio
realm discover dominio.local
3. Unirse al dominio
sudo realm join dominio.local -U administrador
4. Verificar la unión
Muestra los dominios a los que pertenece el equipo:
realm list
Opcional: OpenLDAP (autenticación centralizada)
sudo apt install slapd ldap-utils
OpenLDAP permite gestionar políticas de seguridad y autenticación centralizada en entornos Linux puros, sin depender de Active Directory.
