6.5 Cortafuegos

Filtra el tráfico entrante y saliente según reglas definidas para proteger el sistema.

📥
Tráfico entrante
Conexiones que llegan al equipo desde la red.
📤
Tráfico saliente
Conexiones que salen del equipo hacia la red.
📋
Reglas
Definen qué puertos, IPs o apps se permiten o bloquean.

PowerShell — reglas avanzadasCLI
Abrir un puerto (entrante)
New-NetFirewallRule ` -DisplayName "Abrir Puerto 8080" ` -Direction Inbound ` -Protocol TCP ` -LocalPort 8080 ` -Action Allow
Bloquear una aplicación (saliente)
New-NetFirewallRule ` -DisplayName "Bloquear App" ` -Program "C:\Ruta\App.exe" ` -Direction Outbound ` -Action Block
Direction: Inbound = tráfico entrante  ·  Outbound = tráfico saliente
Action: Allow = permitir  ·  Block = bloquear

Ubuntu UFW e iptables

UFW — comandos esencialesCLI
Activar y estado
sudo ufw enableActiva el cortafuegos
sudo ufw disableDesactiva el cortafuegos
sudo ufw status numberedVer todas las reglas activas
Permitir y bloquear
sudo ufw allow sshPermitir SSH (puerto 22)
sudo ufw allow httpPermitir HTTP (puerto 80)
sudo ufw allow httpsPermitir HTTPS (puerto 443)
sudo ufw deny httpBloquear HTTP
Gestión de reglas
sudo ufw delete [nº regla]Eliminar una regla por número
UFW avanzado e iptablesCLI
Reglas avanzadas con UFW
Permitir rango de puertos:
sudo ufw allow 1000:2000/tcp
Permitir SSH solo desde una IP:
sudo ufw allow from 192.168.1.10 to any port 22
iptables — control granular
Bloquear todo el tráfico entrante:
sudo iptables -P INPUT DROP
Excepto SSH:
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables es más potente pero complejo. UFW es una capa simplificada sobre él, recomendada para la mayoría de casos.
Ejemplo: proteger un servidor web
sudo ufw allow http
sudo ufw allow https
sudo ufw enable
sudo ufw status
