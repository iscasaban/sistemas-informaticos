6.4 Permisos de red

Controla quién puede leer, modificar o eliminar recursos compartidos en la red.

Niveles de permiso en carpetas compartidas

👁️
Leer
Solo visualizar archivos. Sin modificar ni eliminar.
✏️
Cambiar
Modificar archivos, pero no eliminar la carpeta compartida.
🔑
Control total
Leer, escribir, ejecutar y eliminar sin restricciones.

Windows Gestión de permisos de red

Permisos básicosGUI
1
Clic derecho en la carpeta → Propiedades → Compartir → Compartir avanzado
2
Activa Compartir esta carpeta → clic en Permisos
3
Selecciona usuario o grupo y asigna: Leer / Cambiar / Control total
Permisos avanzadosGUI
1
Propiedades → pestaña Seguridad → Editar
2
Selecciona un usuario o grupo
3
Define permisos específicos: Lectura y ejecución / Modificar / Control total
Permisos por línea de comandosCLI
icacls — gestión de permisos NTFS
icacls "C:\CarpetaCompartida" /grant Usuario:(F)
Valores de permiso:
(F) = Control total  ·  (M) = Modificar  ·  (R) = Leer  ·  (W) = Escribir
Políticas de grupo (entornos empresariales)
gpedit.mscAbre el editor de políticas locales
Ruta: Configuración del equipo → Configuración de Windows → Seguridad → Directivas locales

Ubuntu Gestión de permisos de red

Permisos con SambaCLI
Configurar recurso compartido
[Compartida] path = /home/usuario/compartida read only = no browsable = yes
Permisos del sistema de archivos
sudo chmod 770 /home/usuario/compartida
sudo chown usuario:grupo /home/usuario/compartida
Añadir usuario a Samba y reiniciar
sudo smbpasswd -a usuario
sudo systemctl restart smbd
chmod y chownCLI
¿Qué significa chmod 770?
7
Propietario
rwx (todo)
7
Grupo
rwx (todo)
0
Otros
sin acceso
Valores numéricos de chmod
4= Lectura (r)
2= Escritura (w)
1= Ejecución (x)
7= r+w+x (4+2+1 = todo)
AppArmor — refuerzo adicional
sudo aa-statusVer estado de AppArmor
Restringe qué puede hacer cada aplicación sobre los recursos compartidos, aunque tenga permisos del sistema de archivos.
